Why muxwarp

No config duplication. No session creation. Just fast remote tmux attachment.

⚡

Parallel scanning

All hosts are scanned concurrently. Sessions stream into the TUI as each host responds — no waiting for the slowest one.

🔎

Fuzzy filter

Press / and start typing. Fuzzy matching across host and session names with character-level highlighting.

🔒

Secure by design

No shell interpolation in SSH commands. Session names validated against strict allowlist. Args passed directly to execve.

🔌

Uses your ssh

Calls your system ssh binary directly. All your SSH config, agent forwarding, ProxyCommand, FIDO keys — everything works.

♻

Reconnection loop

After ssh exits, you're back in the TUI with a fresh scan. Detach from tmux and pick another session instantly.

🎨

Retro-futurism TUI

Neon gradient header, column-aligned sessions, warp animation with color gradient bar. Built on Bubble Tea v2.

How it works

1

Configure your hosts

List your SSH targets in ~/.muxwarp.config.yaml. Same strings you'd pass to ssh.

2

Scan in parallel

muxwarp runs ssh ... tmux list-sessions on every host concurrently. Results stream in as each host responds.

3

Pick a session

Navigate the list, use fuzzy filter to search, or run muxwarp <name> to jump straight in.

4

Warp

A brief animation plays, then your process is replaced with ssh -t host -- tmux attach. Clean TTY handoff, no orphaned parent.

Simple config

One file. No env vars, no XDG, no merging.

# ~/.muxwarp.config.yaml
defaults:
  timeout: 3s               # per-host SSH timeout
  term: xterm-256color    # TERM on attach

hosts:
  - clint@indigo
  - clint@devbox
  - deploy-host           # SSH config aliases work
